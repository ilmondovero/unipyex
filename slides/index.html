ES03 – Confronto col Benchmark

Aggiungi MSCI World e FTSE MIB come benchmark
import pandas as pd

BENCHMARK = {
    "MSCI_World": "IWDA.AS",   # iShares MSCI World ETF
    "FTSE_MIB":   "FTSEMIB.MI" # ETF su indice italiano
}

# Download benchmark
bench_raw = yf.download(
    list(BENCHMARK.values()), period="5y", progress=False
)
bench = bench_raw["Adj Close"]
bench.columns = list(BENCHMARK.keys())

# Unisci con il portafoglio
tutto = pd.concat([prezzi, bench], axis=1)

# Allinea le date (dropna rimuove gg in cui manca almeno un valore)
tutto.dropna(inplace=True)
print(f"Date comuni: {len(tutto)}")
pd.concat(axis=1) affianca DataFrame per colonna. axis=0 li impila per riga. dropna() con default how='any' rimuove righe con almeno un NaN. Importante per avere serie allineate quando si confrontano mercati diversi (es. festivi italiani vs USA).

Normalizzazione a Base 100

Per confrontare titoli con prezzi assoluti diversi (Ferrari ~350€, Microsoft ~400$) occorre normalizzare.

Formula

Pbase100 = (Pt / P0) × 100
  • P0 = primo prezzo della serie
  • Pt = prezzo al giorno t
  • Tutti partono da 100
  • Valore 115 = +15% dal inizio
  • Valore 85 = –15% dal inizio

Codice

# Normalizza: primo valore = 100
normalizzato = (
    tutto / tutto.iloc[0] * 100
)

print(normalizzato.tail(3))
# Ferrari   Microsoft  MSCI_W
# 134.2     189.5      121.3
# ...
tutto.iloc[0] = prima riga (divisore).
L'operazione si applica automaticamente a tutte le colonne (broadcasting).
La normalizzazione è fondamentale in finanza per confrontare asset con scale diverse. Chiedere: "Se Ferrari vale 350€ e Microsoft 400$, chi ha reso di più?" Non si può rispondere senza normalizzare. La base 100 risolve questo problema.

ES04 – Rendimenti e Volatilita

import numpy as np

# Rendimenti giornalieri (variazione % rispetto al giorno prima)
rend_giorn = prezzi.pct_change().dropna()

# Statistiche annualizzate (252 = gg di borsa in un anno)
TRADING_DAYS = 252

rendimento_annuo = rend_giorn.mean() * TRADING_DAYS
volatilita_annua = rend_giorn.std() * np.sqrt(TRADING_DAYS)

print("=== RIEPILOGO PORTAFOGLIO ===")
for ticker in prezzi.columns:
    r = rendimento_annuo[ticker]
    v = volatilita_annua[ticker]
    print(f"{ticker:12s}: rendimento {r:+.1%}  |  volatilità {v:.1%}")
Eseguire in Jupyter e discutere: chi offre il miglior rapporto rischio/rendimento?
pct_change() calcola (P_t - P_t-1) / P_t-1 per ogni riga. L'annualizzazione moltiplica media per 252 (gg di trading) e std per sqrt(252) perché varianza si somma linearmente ma deviazione standard scala con radice.

Sharpe Ratio

Formula

Sharpe = (Rp − Rf) / σp
  • Rp = rendimento del portafoglio
  • Rf = tasso risk-free (BCE ~3.5%)
  • σp = volatilita annua
  • Sharpe > 1 = buono
  • Sharpe > 2 = eccellente
  • Sharpe < 0.5 = rischioso
RISK_FREE = 0.035  # BCE 2024

sharpe = (
    (rendimento_annuo - RISK_FREE)
    / volatilita_annua
)

# Ranking dal migliore
ranking = sharpe.sort_values(
    ascending=False
)
print("\nRanking Sharpe Ratio:")
for t, s in ranking.items():
    emoji = "✓" if s > 1 else "~"
    print(f"  {emoji} {t:12s}: {s:.2f}")
Il Sharpe ratio misura la qualità del rendimento: quante unità di rischio prendo per ogni unità di rendimento sopra al tasso risk-free. Un investimento con Sharpe = 1.5 è preferibile a uno con Sharpe = 0.8, anche se il secondo ha rendimento assoluto maggiore.

Matrice di Correlazione

# Correlazione tra rendimenti giornalieri
corr = rend_giorn.corr()

# Formatta output
print("Matrice Correlazione:")
print(corr.round(2).to_string())

Interpretazione

  • 1.0 = movimento identico
  • 0.0 = nessuna relazione (ideale per diversificare)
  • −1.0 = movimento opposto
  • Ferrari & LVMH spesso correlate (entrambe luxury)
  • Terna (utilities) poco correlata con tech USA
Diversificazione: un portafoglio ben diversificato combina asset con correlazione bassa (< 0.5). La Famiglia Bianchi ha titoli europei e USA, utilities e tech: buona diversificazione geografica e settoriale.
Notare che MSFT e GOOGL hanno correlazione ~0.85: entrambe big-tech USA, si muovono spesso insieme.
La correlazione è la misura statistica della co-variazione tra due serie. In gestione di portafoglio, Markowitz dimostrò nel 1952 che combinare asset poco correlati riduce il rischio senza necessariamente ridurre il rendimento.

  • Applica formule, formattazione, protezione

xlwings usa il protocollo COM su Windows (o AppleScript su Mac) per comunicare con l'applicazione Excel aperta. Questo significa che TUTTE le funzionalità di Excel sono disponibili — incluse quelle che openpyxl non supporta (grafici reali, protezione, VBA).

ES06 – DataFrame in Excel

Scrivi la tabella prezzi in un foglio Excel
import xlwings as xw

# Apri Excel (visible=True lo rende visibile sullo schermo)
app = xw.App(visible=True)
wb  = app.books.add()
ws  = wb.sheets[0]
ws.name = "Prezzi"

# Scrivere un DataFrame: una sola riga!
ws.range("A1").value = prezzi

# Scrivi anche l'intestazione
ws.range("A1").value = [prezzi.columns.tolist()]  # header
ws.range("A2").options(index=True).value = prezzi  # dati + indice

# Salva
wb.save("output/report_bianchi.xlsx")
print("File salvato!")
.options(index=True) include la colonna data come prima colonna. Senza index=False, l'indice viene sempre scritto.
Questa singola riga sostituisce settimane di copia-incolla. xlwings serializza automaticamente il DataFrame: converte le date in formato Excel, i float in numeri, i NaN in celle vuote. La velocità dipende dalle dimensioni: 1258 righe x 7 colonne richiede meno di 1 secondo.

La trappola del NumberFormat

In Excel italiano, il punto . e la virgola , hanno significato INVERTITO rispetto all'inglese!

SBAGLIATO

# Excel inglese — NON fare in italiano!
ws.range("B2").number_format = "0.0%"
# Risultato: "04%" invece di "3,5%"
# Excel legge il punto come migliaia!

ws.range("C2").number_format = "#,##0"
# Risultato: "3000000,0"
# La virgola viene letta come decimale!

CORRETTO

# SEMPRE usare NumberFormatLocal con
# sintassi italiana:
ws.range("B2").api.NumberFormatLocal = "0,0%"
# Virgola = decimale in IT → OK!

ws.range("C2").api.NumberFormatLocal = "#.##0"
# Punto = migliaia in IT → OK!

# Esempi pratici:
# "+0,0%;-0,0%"  rendimento con segno
# "#.##0,00 €"   valuta italiana
# "gg/mm/aaaa"   data in italiano
QUESTO E' IL BUG PIU' COMUNE! Ogni studente lo incontra la prima volta. In inglese: punto = decimale, virgola = migliaia. In italiano (e nella maggior parte d'Europa): virgola = decimale, punto = migliaia. Excel rispetta il locale del sistema operativo. NumberFormatLocal usa la sintassi del locale corrente; NumberFormat usa sempre la sintassi inglese.

ES08 – Il problema delle formule con @

Excel 365 aggiunge automaticamente @ davanti alle formule "legacy":

Con FormulaLocal

# xlwings usa FormulaLocal → Excel aggiunge @
ws.range("B5").api.FormulaLocal = \
    "=SOMMA(D13:D22)"

# In Excel la cella mostra:
# =@SOMMA(D13:D22)
# Gli studenti si confondono!

Con Formula2Local

# Formula2Local = dynamic array → no @
ws.range("B5").api.Formula2Local = \
    "=SOMMA(D13:D22)"

# In Excel la cella mostra:
# =SOMMA(D13:D22)
# Pulito e corretto!
Formula2Local e disponibile da Excel 365 (build 2009+). Per versioni precedenti fare fallback su FormulaLocal.
Il simbolo @ in Excel 365 indica "implicit intersection operator": un meccanismo per compatibilità con le vecchie formule single-value in un mondo di dynamic arrays. Gli studenti vedono @ e pensano ci sia un errore. Formula2Local evita il problema usando direttamente la modalità dynamic array.

La Funzione set_formula()

In tw_utils.py — usarla SEMPRE per impostare formule:

def set_formula(rng, formula_it):
    """
    Imposta una formula italiana senza @ in Excel 365.

    Args:
        rng:        range xlwings (es. ws.range("B5"))
        formula_it: formula in italiano con = iniziale
                    (es. "=SOMMA(D13:D22)")
    """
    try:
        # Excel 365+: dynamic array, no @
        rng.api.Formula2Local = formula_it
    except AttributeError:
        # Excel 2019 e precedenti: fallback
        rng.api.FormulaLocal = formula_it


# Utilizzo:
from tw_utils import set_formula

set_formula(ws.range("B5"), "=SOMMA(D13:D22)")
set_formula(ws.range("C5"), "=MEDIA(D13:D22)")
set_formula(ws.range("D5"), "=CONTA.SE(E13:E22;\"OK\")")
Il pattern try/except gestisce la compatibilità tra versioni di Excel. Formula2Local esiste da Excel 365; su versioni precedenti lancia AttributeError e si fa fallback su FormulaLocal (che aggiungerà @ ma almeno funziona). ENFATIZZARE: usare sempre set_formula(), mai assegnare .value = "=..." per formule.

Il Punto e Virgola Italiano

Excel italiano usa ; come separatore argomenti, non ,

Funzione Italiano (CORRETTO) Inglese (SBAGLIATO)
Condizione =SE(A1>10;"Grande";"Piccolo") =IF(A1>10,"Grande","Piccolo")
Somma.Se =SOMMA.SE(A1:A10;">5") =SUMIF(A1:A10,">5")
Cerca verticale =CERCA.VERT(B2;$D:$E;2;FALSO) =VLOOKUP(B2,$D:$E,2,FALSE)
Testo formula =TESTO.FORMULA(B5) =FORMULATEXT(B5)
Conta.Se =CONTA.SE(D1:D50;"OK") =COUNTIF(D1:D50,"OK")

Questo errore capita a tutti, anche ai professionisti, quando si passa da documentazione inglese (Stack Overflow, Microsoft Docs) a Excel italiano. Il cambio separatore vale anche per gli argomenti in formule come CERCA.VERT, SE, SOMMA.SE, ecc. Negli script Python le formule le scriviamo in italiano perché Excel le eseguirà nel locale italiano dell'utente.

Tipi di Grafico – Quando Usarli

Tipoxlwings codeUso ideale in finanza
Line "line" Prezzi storici, performance nel tempo, confronto benchmark
Bar / Column "bar" / "column_clustered" Confronto rendimenti annuali, asset allocation, TER per ETF
Pie / Donut "pie" / "doughnut" Composizione portafoglio (% per asset class / geo)
Scatter "xy_scatter" Rischio vs Rendimento (frontiera efficiente), correlazioni
Il grafico a dispersione (scatter) Rischio/Rendimento e il piu usato nella presentazione di portafogli professionali: ogni bolla e un titolo, asse X = volatilita, asse Y = rendimento.
Regola empirica in finanza: per tutto ciò che cambia nel tempo, usare line chart. Per confronti statici (chi rende di più), bar chart. Per composizioni, pie/donut. Il bubble chart (variante dello scatter con dimensione della bolla = peso nel portafoglio) è la versione professionale della frontiera di Markowitz.

Schema di protezione

Cella: Locked = True
Non modificabile quando il foglio e protetto
Cella: FormulaHidden = True
Formula invisibile nella barra — SOLO se foglio protetto
Foglio: Protect(password)
Attiva Locked e FormulaHidden su tutte le celle
L'errore più comune con la protezione: impostare FormulaHidden = True ma dimenticarsi di chiamare .Protect() sul foglio. Senza protezione, FormulaHidden non ha effetto. Sempre proteggere DOPO aver configurato le celle.

ES13 – Export PDF Professionale

def esporta_pdf(wb, output_path: str):
    """Esporta il workbook come PDF."""
    for ws in wb.sheets:
        if ws.name.startswith("_"): continue
        setup = ws.api.PageSetup
        setup.Orientation = 2           # landscape
        setup.FitToPagesWide = 1
        setup.CenterHorizontally = True

    wb.api.ExportAsFixedFormat(
        Type=0, Filename=os.path.abspath(output_path),
        Quality=0, OpenAfterPublish=False)
ExportAsFixedFormat è la funzione COM nativa di Excel per esportare PDF. Supporta tutte le impostazioni di PageSetup. OpenAfterPublish=True aprirebbe automaticamente il PDF dopo la generazione — utile per debug. Questa singola funzione sostituisce Ctrl+P → Salva come PDF manuale.

ES14 – Simulazione vs Realta

Dati simulati (tw_config.py)

import random

def genera_prezzi_simulati(
    start=100, n_giorni=1260, drift=0.0003, vol=0.015
):
    """Moto Browniano Geometrico semplificato."""
    prezzi = [start]
    for _ in range(n_giorni - 1):
        rendimento = random.gauss(drift, vol)
        prezzi.append(prezzi[-1] * (1 + rendimento))
    return prezzi

Dati reali (yfinance)

df = yf.download("RACE.MI", period="5y")
prezzi_reali = df["Adj Close"].tolist()

Limitazioni simulazione

  • Distribuzione normale vs reale (fat tails)
  • Nessuna correlazione tra asset
  • Drift costante (irrealistico)
  • Nessun evento di mercato (COVID, crisi)
I dati reali da yfinance catturano eventi storici reali che la simulazione non puo riprodurre.
Il Moto Browniano Geometrico (GBM) è il modello più semplice per i prezzi azionari, introdotto da Black-Scholes. I prezzi reali mostrano "fat tails" (eventi estremi più frequenti del previsto da una distribuzione normale), mean reversion, clustering di volatilità. Per esercizi didattici la simulazione va bene; per decisioni reali si usano sempre dati storici reali.

import os, pickle
from pathlib import Path

CACHE = Path("cache/prezzi.pkl")

def scarica_con_cache(tickers, period="5y"):
    if CACHE.exists():
        print("Carico da cache...")
        with open(CACHE, "rb") as f:
            return pickle.load(f)
    try:
        df = yf.download(tickers,
                         period=period,
                         progress=False)
        CACHE.parent.mkdir(exist_ok=True)
        with open(CACHE, "wb") as f:
            pickle.dump(df, f)
        return df
